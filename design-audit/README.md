# CrossSync 产品体验审计（2026-07-16）

## 审计范围

- 产品：CrossSync 局域网文件传输
- 核心目标：让用户把 iPhone 照片可靠、直接地传到电脑，并清楚知道连接、准备、上传和完成状态
- 已检查：首次扫码连接页、桌面工作台、390px 手机网页布局、文件管理抽屉，以及 SwiftUI 原生客户端的五种状态实现
- 证据限制：本机未安装完整 Xcode，无法运行 iOS 模拟器；原生 App 的判断来自代码结构，不作为截图证据。没有执行真实文件上传，以免改变现有数据。

## 总体结论

功能已经比较完整，真正需要重做的不是“皮肤”，而是信息架构。当前网页首屏同时展示选择、三个传输阶段、预计时间、常亮、连接、安全、保存位置、反向发送和文件管理，导致最重要的动作“选择并发送”反而不够聚焦。建议把产品重构为“一个主任务 + 渐进展开状态 + 独立文件管理”的三层结构。

## 流程步骤与健康度

### 1. 扫码连接 — 需要改进

![首次连接页](/tmp/crosssync-connection-qr-audit.png)

- 优点：二维码、局域网要求和打开工作台入口都在同一页，任务入口明确。
- 风险：完整 URL 和访问令牌并列展示，既造成视觉噪音，也让“复制哪个信息”不清楚；窄屏下 URL 被逐字换行。
- 建议：二维码作为唯一主动作；把“原生 App 手动连接”收进次级区域，只显示电脑地址、令牌和复制按钮，不直接展示完整带参数 URL。

### 2. 桌面工作台空闲态 — 结构过载

![桌面工作台](/Users/ryanzhao/Ryan/CrossSync/design-audit/screenshots/01-desktop-workbench.png)

- 优点：设备两端、保存目标和局域网直连的信任信息表达完整。
- 风险：空闲时就展示“照片准备 / 等待传输 / 上传中”三个全尺寸零状态模块，用户需要先理解系统内部管线，才能开始一个很简单的任务。
- 风险：`HTTP 降级模式` 与底部 `安全传输中` 同时出现，信任信号彼此矛盾。
- 建议：空闲态只保留一个主动作、目标电脑和保存位置摘要。传输开始后再出现一条四阶段 stepper，并把速度、剩余时间和当前文件放在同一个进度区域。

### 3. 手机网页工作台 — 可用但过长

![手机网页工作台](/Users/ryanzhao/Ryan/CrossSync/design-audit/screenshots/02-mobile-web-workbench.png)

- 优点：390px 下没有水平溢出，主要控件仍能阅读和点击。
- 风险：空闲页高度约 2270px；开始操作前就要经过三个状态卡、常亮说明、目标电脑、安全条等大量内容。
- 风险：关键状态依赖青、紫、粉、绿区分，文字和形状虽然提供了部分补充，但整体仍偏“仪表盘”，不够像一次线性任务。
- 建议：手机端使用单列任务流：顶部连接状态；中间选取按钮；底部保存目标。只有上传期间固定显示进度和取消/重试操作。

### 4. 文件管理抽屉 — 当前存在严重可读性缺陷

![文件管理抽屉](/Users/ryanzhao/Ryan/CrossSync/design-audit/screenshots/04-file-management-drawer.png)

- 已确认问题：`.queue-panel` 和 `.file-section` 的背景为近白色，但继承文字颜色为近白色；标题和列表几乎不可见。
- 已确认问题：选项标签使用深色背景但文字颜色来自旧的浅色主题变量，出现主题规则互相覆盖。
- 风险：删除、清空、下载、打开目录等高风险动作集中在同一层级，缺少选择上下文和破坏性操作分组。
- 建议：文件管理从主工作台拆成独立视图；统一暗色 token；默认只显示刷新、打开和选择，选中项目后才显示下载/删除批量操作。

## 最高优先级修改

1. 修复文件管理抽屉的主题冲突和不可读文字，这是当前最明确的可用性缺陷。
2. 重构首屏信息架构：空闲态不展示三个零状态传输卡。
3. 统一 Web 与 iOS 的状态模型和文案：连接 → 选择 → 准备 → 传输 → 完成/失败。
4. 消除安全文案矛盾：HTTP 下应明确“局域网内传输，但连接未加密”，HTTPS 才显示“加密连接”。
5. 把反向发送与文件管理移出主任务首屏，作为清晰的次级入口。
6. 用统一图标库替代当前的文字符号和 emoji，并验证键盘焦点、动态状态播报、触控目标和颜色对比度。

## 建议的产品结构

- **连接页**：扫码；原生 App 手动连接（折叠）；HTTPS 证书引导（按需出现）。
- **传输首页**：连接状态、发送目标、一个“选择照片或文件”主动作、保存位置摘要。
- **进行中**：单一总进度、当前阶段、速度/剩余时间、当前文件、暂停或取消。
- **完成页**：成功数量、失败数量、保存位置、再次传输、打开文件夹。
- **文件管理**：电脑接收区与 iPhone 共享箱独立页面，不与主传输任务竞争注意力。
